### 📝 Introduction
> "Design is not just what it looks like and feels like. Design is how it works."

Human-Computer Interaction (HCI) is the study of how people interact with computers and to what extent computers are or are not developed for successful interaction with human beings. For an aspiring Data Engineer, HCI is vital because even the most powerful data system is useless if the interface for the end-user is unintuitive. This course focuses on making technology accessible, efficient, and user-centered.

## 📚 Syllabus & Topics
Here is the breakdown of the chapters we covered this semester:

| Topic | Description | Key Concepts |
| :---: | :--- | :--- |
| 1️⃣ | **Intro to HCI** | Definition, goals, and the evolution of interfaces. |
| 2️⃣ | **Cognition & Social Mechanism** | How users think, feel, and collaborate in digital spaces. |
| 3️⃣ | **Conceptualization & Trends** | Interaction design paradigms and emerging UI trends. |
| 4️⃣ | **Interaction Design Process** | The iterative lifecycle of creating user-centered designs. |
| 5️⃣ | **Data Gathering & Personas** | Identifying user needs and defining target user profiles. |
| 6️⃣ | **Prototyping & Gestalt** | Low/High-fidelity prototypes and UI design principles. |
| 7️⃣ | **Evaluation** | Testing systems for usability and user satisfaction. |
| 8️⃣ | **Predictive Models & Analytics** | Using data to predict user behavior and performance. |
| 9️⃣ | **Analysis & Interpretation** | Processing user data to present design solutions. |

## 💭 Student Reflection

### 💡 Key Highlights
* **🧠 User Psychology:** Understanding *Cognition and Emotion* was fascinating. It taught me that a good developer must anticipate a user's mental model to prevent frustration.
* **🎨 Gestalt Principles:** Learning how the human eye perceives groups and patterns has significantly improved my ability to design clean, organized dashboards.

### 📈 Suggestions for Improvement
* *Hands-on Prototyping:* I would enjoy more intensive workshops using professional design tools like **Figma** or **Adobe XD** to bring our conceptual designs to life.

### 🚀 Reflection
HCI has taught me to look at software through the eyes of the user. In the field of Data Engineering, this means building data visualizations and reporting tools that are not just technically accurate, but also easy for stakeholders to understand. It's about bridging the gap between complex data and human understanding.
