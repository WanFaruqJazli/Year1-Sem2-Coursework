### 📝 Introduction
> "First, solve the problem. Then, write the code."

Programming Technique II marks my shift from procedural programming to the **Object-Oriented Programming (OOP)** paradigm. This course is about building scalable, reusable, and efficient software. As a Data Engineering student, mastering these concepts is vital for developing complex data processing systems and understanding how modern software frameworks are architected.

---

## 📚 Syllabus & Topics
Here is the breakdown of the chapters we covered this semester:

| Chapter | Topic | Key Concepts |
| :---: | :--- | :--- |
| 1️⃣ | **Intro to OOP & Software Dev** | Software life cycle, modularity, and the OOP mindset. |
| 2️⃣ | **Classes & Objects** | Encapsulation, data members, and member functions. |
| 3️⃣ | **Constructors & Destructors** | Object initialization, memory management, and cleanup. |
| 4️⃣ | **Object Manipulations** | Array of objects, passing objects to functions, and pointers. |
| 5️⃣ | **Relationship Concepts** | Association, Aggregation, and Composition. |
| 6️⃣ | **Inheritance** | Base/Derived classes, reusability, and access specifiers. |
| 7️⃣ | **Polymorphism** | Virtual functions, abstract classes, and dynamic binding. |
| 8️⃣ | **Advanced C++ Features** | Templates, Exception Handling, and File Processing. |

## 💭 Student Reflection

### 💡 Key Highlights
* **🏗️ Modularity:** Learning about *Classes* changed how I organize my code. Instead of a long list of functions, I now think in terms of "Entities" that have their own behaviors and data.
* **🧬 Reusability:** *Inheritance* and *Polymorphism* showed me how to write code once and extend it for different purposes, which is a massive time-saver for large-scale projects.

### 📈 Suggestions for Improvement
* *Design Patterns:* While we focus on the syntax of OOP, I’d love to dive deeper into *Design Patterns* to see how professional engineers structure their class hierarchies for maximum efficiency.

### 🚀 Reflection
Moving beyond basic C++ has been challenging but rewarding. Understanding the relationships between objects (like Aggregation vs. Composition) has made my logic much cleaner. I can see how these OOP principles will eventually help me build robust data pipelines and tools in the future.
